# 파트 6: 대회 — 최고의 AI 시인

Shakespeare로 GPT를 학습했습니다. 이제 배운 것을 실제로 써먹을 차례입니다. 가능한 최고의 시 생성 모델을 학습하고, 최고의 시를 제출하세요.

## 도전 과제

가장 좋은 시를 생성하는 모델을 학습하세요. 데이터셋, 모델 크기, 토크나이저, 학습 전략은 모두 직접 고를 수 있습니다. 다음 조건만 지키면 무엇이든 가능합니다.

1. 모델을 직접 학습해야 합니다(사전학습 가중치 금지, 다른 사람 모델 fine-tuning 금지)
2. 시는 모델이 생성해야 하며, 직접 쓰거나 편집하면 안 됩니다
3. 증명할 수 있어야 합니다. 체크포인트와 정확한 생성 명령을 제출하세요

## 제출

세 가지를 제출합니다.

1. **시** — 모델의 가장 좋은 출력
2. **체크포인트** — 재현할 수 있도록 제출
3. **생성 명령** — 사용한 정확한 프롬프트, temperature, top_k, seed

생성을 재현 가능하게 만들려면 `--seed` 플래그를 사용하세요. 같은 seed + 같은 체크포인트 + 같은 인자는 항상 같은 출력을 만듭니다.

```bash
# 제출 명령 예시
python generate.py checkpoint_best.pt --prompt "The morning sun" --temperature 0.7 --top_k 30 --seed 42
```

우리는 이 정확한 명령을 체크포인트에 대해 실행해 제출물을 검증합니다.

## 이기는 방법

우승 시는 다음 기준으로 평가합니다.

- **일관성** — 시로서 말이 되는가?
- **창의성** — 흥미로운 이미지, 예상 밖의 표현이 있는가?
- **구조** — 리듬, 줄바꿈, 연이 있는가?
- **감정** — 어떤 느낌을 불러일으키는가?

아무도 GPT-2 수준의 품질을 기대하지 않습니다. 기준은 "분명 기계가 썼지만 노력하고 있다는 것이 보인다"입니다.

## 탐구할 아이디어

기초는 배웠습니다. 이제 더 밀어붙이세요. 조정할 수 있는 레버는 다음과 같습니다.

### 더 좋은 데이터

Shakespeare도 동작하지만 하나의 스타일일 뿐입니다. 더 좋은 시 데이터셋을 찾거나 만드세요.

- **Poetry Foundation** — 여러 스타일과 시대의 수천 편의 시
- **Project Gutenberg** — 퍼블릭 도메인 시집
- **여러 출처 결합** — Shakespeare + sonnet + 현대 자유시
- **강하게 선별하기** — 나쁘거나 관련 없는 텍스트를 제거하고 실제 시만 남기기

데이터가 많으면 도움이 되지만, **양보다 품질이 더 중요합니다**. 잘 선별한 시 1MB가 시가 섞인 무작위 텍스트 10MB보다 낫습니다.

### 더 큰 모델

데이터가 더 많으면 바로 과적합하지 않고 더 큰 모델을 정당화할 수 있습니다.

| 설정 | 파라미터 | n_layer | n_head | n_embd |
|------|----------|---------|--------|--------|
| 워크숍 기본값 | ~10M | 6 | 6 | 384 |
| 더 큼 | ~25M | 8 | 8 | 512 |
| GPT-2 Small | ~85M | 12 | 12 | 768 |

과적합에서 배운 점을 기억하세요. 같은 데이터에 더 큰 모델을 쓰면 더 빨리 외울 뿐입니다. 모델은 데이터와 함께 키워야 합니다.

### 더 좋은 토크나이저

문자 단위 방식은 데이터셋이 작았기 때문에 Shakespeare에 잘 맞았습니다. 더 큰 시 데이터셋을 쓴다면 다음을 고려하세요.

- **tiktoken을 사용한 BPE** — GPT-2 토크나이저, 약 5MB 이상 데이터셋에 좋음
- **직접 BPE 학습** — 시 어휘에 맞춘 더 작은 어휘(1k-5k 토큰)
- **단어 단위** — 데이터셋이 충분히 크고 모델이 단어 단위로 생각하길 원할 때

### 학습 조정

- **더 긴 컨텍스트**(`block_size=512` 또는 `1024`) — 모델이 여러 줄에 걸친 전체 연과 압운 구조를 잡게 합니다
- **Dropout** — attention과 MLP 블록에 `nn.Dropout(0.1)`을 추가해 과적합을 줄입니다
- **Early stopping** — 마지막 체크포인트가 아니라 val loss가 가장 낮은 체크포인트를 저장합니다
- **학습률** — 특정 모델/데이터 조합에 맞게 `max_lr`을 조정합니다

### 생성 기법

같은 모델도 생성 설정에 따라 매우 다른 출력을 만들 수 있습니다.

- **Temperature** — 0.6부터 1.0까지 시도하세요. 낮을수록 더 집중된 출력이 나옵니다
- **Top-k** — 10부터 50까지 시도하세요. 낮을수록 더 가능성 높은 토큰으로 제한됩니다
- **프롬프트 엔지니어링** — 프롬프트가 중요합니다. 제목, 첫 줄, 스타일 힌트로 시작해 보세요
- **Cherry-pick** — 시 20편을 생성하고 가장 좋은 것을 제출하세요. 이것은 허용됩니다.
